## ADDED Requirements

### Requirement: Timer always visible at bottom

Система ДОЛЖНА отображать таймер Помодоро в нижней части экрана постоянно, независимо от навигации между разделами.

#### Scenario: Timer persists during navigation

- **WHEN** пользователь переключается между видами задач или разделами приложения
- **THEN** таймер остаётся видимым в нижней части и продолжает работать

#### Scenario: Timer state preserved on page reload

- **WHEN** пользователь перезагружает страницу во время работы таймера
- **THEN** таймер восстанавливает своё состояние (текущая задача, оставшееся время, тип интервала)

### Requirement: Start timer on task

Система ДОЛЖНА позволять запустить таймер с привязкой к конкретной задаче.

#### Scenario: Start timer with task selected

- **WHEN** пользователь выбрал задачу и нажимает кнопку "Пуск" на таймере
- **THEN** таймер начинает отсчёт рабочего интервала для выбранной задачи

#### Scenario: Start timer without task shows selection dialog

- **WHEN** пользователь нажимает "Пуск" без выбранной задачи
- **THEN** система отображает диалог со списком задач на сегодня для выбора

#### Scenario: Timer displays active task title

- **WHEN** таймер работает на задаче
- **THEN** интерфейс таймера отображает название текущей задачи

### Requirement: Pause timer

Система ДОЛЖНА позволять поставить таймер на паузу.

#### Scenario: Pause running timer

- **WHEN** пользователь нажимает кнопку "Пауза" во время работы таймера
- **THEN** таймер останавливает отсчёт и сохраняет текущее состояние

#### Scenario: Resume paused timer

- **WHEN** пользователь нажимает "Пуск" на паузированном таймере
- **THEN** таймер продолжает отсчёт с момента паузы

#### Scenario: Paused work interval creates time segment

- **WHEN** пользователь ставит на паузу рабочий интервал
- **THEN** система фиксирует отрезок времени от начала работы до паузы

### Requirement: Skip to next interval

Система ДОЛЖНА позволять досрочно завершить текущий интервал и перейти к следующему.

#### Scenario: Skip work interval

- **WHEN** пользователь нажимает "Следующий" во время рабочего интервала
- **THEN** система завершает интервал досрочно, фиксирует время, и переходит к следующему интервалу в последовательности

#### Scenario: Skip break interval

- **WHEN** пользователь нажимает "Следующий" во время интервала отдыха
- **THEN** система переходит к следующему интервалу без фиксации времени

### Requirement: Configurable interval sequence

Система ДОЛЖНА позволять настраивать последовательность интервалов (работа/отдых, длительность, порядок).

#### Scenario: Default pomodoro sequence

- **WHEN** пользователь впервые запускает систему
- **THEN** таймер использует классическую последовательность: 25 мин работа, 5 мин отдых, 25 мин работа, 5 мин отдых, 25 мин работа, 5 мин отдых, 25 мин работа, 15 мин длинный отдых

#### Scenario: Customize interval duration

- **WHEN** пользователь изменяет длительность рабочего интервала на 50 минут
- **THEN** следующий рабочий интервал длится 50 минут

#### Scenario: Customize interval type

- **WHEN** пользователь добавляет новый интервал типа "отдых" в последовательность
- **THEN** таймер включает этот интервал в цикл

#### Scenario: Reorder intervals

- **WHEN** пользователь изменяет порядок интервалов в настройках
- **THEN** таймер следует новому порядку при следующем цикле

### Requirement: Post-work interval questions

Система ДОЛЖНА задавать два вопроса после завершения каждого рабочего интервала.

#### Scenario: Ask about task progress

- **WHEN** рабочий интервал завершается
- **THEN** система отображает вопрос "Задача продвинулась?" с вариантами: Да / Нет / Застрял

#### Scenario: Ask about energy level

- **WHEN** пользователь ответил на вопрос о прогрессе
- **THEN** система отображает вопрос "Какой был уровень энергии?" с вариантами: Высокий / Средний / Низкий

#### Scenario: Skip questions

- **WHEN** пользователь нажимает "Пропустить" на вопросах
- **THEN** система закрывает диалог без сохранения ответов

#### Scenario: Save answers with time segment

- **WHEN** пользователь отвечает на оба вопроса
- **THEN** система сохраняет ответы в записи time_segment для этого интервала

### Requirement: Interval completion notification

Система ДОЛЖНА уведомлять пользователя о завершении интервала.

#### Scenario: Sound notification on work interval end

- **WHEN** рабочий интервал завершается
- **THEN** система воспроизводит звуковое уведомление для завершения работы (если включено в настройках)

#### Scenario: Sound notification on break interval end

- **WHEN** интервал отдыха завершается
- **THEN** система воспроизводит звуковое уведомление для завершения отдыха (если включено в настройках)

#### Scenario: Visual notification

- **WHEN** интервал завершается
- **THEN** таймер визуально сигнализирует об окончании (изменение цвета, мигание)

### Requirement: Timer countdown display

Система ДОЛЖНА отображать оставшееся время текущего интервала.

#### Scenario: Display time in MM:SS format

- **WHEN** таймер работает
- **THEN** система отображает оставшееся время в формате ММ:СС

#### Scenario: Show interval type indicator

- **WHEN** таймер работает
- **THEN** интерфейс отображает тип текущего интервала (Работа/Отдых) с визуальным индикатором

#### Scenario: Update every second

- **WHEN** таймер активен
- **THEN** отображение времени обновляется каждую секунду

### Requirement: Switch active task during work

Система ДОЛЖНА позволять сменить активную задачу во время рабочего интервала.

#### Scenario: Change task mid-interval

- **WHEN** пользователь выбирает другую задачу во время работы таймера
- **THEN** система фиксирует время на предыдущей задаче и переключает таймер на новую задачу

#### Scenario: Time segment created for previous task

- **WHEN** задача сменяется во время работы
- **THEN** система создаёт запись time_segment для времени работы над предыдущей задачей

### Requirement: Auto-start next interval option

Система ДОЛЖНА предоставлять опцию автоматического запуска следующего интервала.

#### Scenario: Auto-start enabled starts break automatically

- **WHEN** рабочий интервал завершается и опция "Автостарт" включена
- **THEN** система автоматически запускает следующий интервал отдыха через 3 секунды

#### Scenario: Auto-start disabled requires manual start

- **WHEN** интервал завершается и опция "Автостарт" выключена
- **THEN** таймер останавливается и требует нажатия кнопки "Пуск" для следующего интервала
