## ADDED Requirements

### Requirement: Create new task

Система ДОЛЖНА позволять пользователю создавать новую задачу со следующими полями:
- Название (обязательное)
- Описание с форматированием текста и вставкой скриншотов
- Дата, на которую задача запланирована
- Крайний срок (дедлайн)
- Статус
- Принадлежность к проекту
- Принадлежность к клиенту
- Оценка трудоёмкости в помодоро
- Поле "Что конкретно сделать первым"
- Ссылка на задачу во внешней системе

#### Scenario: Create task with minimal fields

- **WHEN** пользователь вводит только название задачи и нажимает "Создать"
- **THEN** система создаёт задачу с указанным названием и значениями по умолчанию для остальных полей

#### Scenario: Create task with all fields

- **WHEN** пользователь заполняет все поля формы создания задачи и нажимает "Создать"
- **THEN** система создаёт задачу со всеми указанными значениями

#### Scenario: Create task without title fails

- **WHEN** пользователь пытается создать задачу без заполнения названия
- **THEN** система отображает сообщение об ошибке "Название задачи обязательно"

### Requirement: Edit existing task

Система ДОЛЖНА позволять пользователю редактировать любое поле существующей задачи.

#### Scenario: Update task title

- **WHEN** пользователь изменяет название задачи и сохраняет изменения
- **THEN** система обновляет название задачи и отображает обновлённое значение

#### Scenario: Update task planned date

- **WHEN** пользователь изменяет дату запланированной задачи
- **THEN** система обновляет дату и задача перемещается в соответствующую колонку на доске по дате

### Requirement: Delete task

Система ДОЛЖНА позволять пользователю удалять задачу.

#### Scenario: Delete task without subtasks

- **WHEN** пользователь удаляет задачу без подзадач
- **THEN** система удаляет задачу и она исчезает из всех видов отображения

#### Scenario: Delete task with subtasks requires confirmation

- **WHEN** пользователь пытается удалить задачу с подзадачами
- **THEN** система запрашивает подтверждение с предупреждением о количестве подзадач

#### Scenario: Confirmed deletion removes task and subtasks

- **WHEN** пользователь подтверждает удаление задачи с подзадачами
- **THEN** система удаляет задачу и все её подзадачи

### Requirement: Create subtask (step)

Система ДОЛЖНА позволять пользователю создавать подзадачи (шаги) для существующей задачи.

#### Scenario: Add step to task

- **WHEN** пользователь добавляет шаг к задаче с указанием названия шага
- **THEN** система создаёт подзадачу связанную с родительской задачей

#### Scenario: Steps inherit parent context

- **WHEN** пользователь создаёт шаг для задачи
- **THEN** шаг автоматически наследует проект и клиента родительской задачи

### Requirement: View task hierarchy

Система ДОЛЖНА отображать иерархическую структуру задач с подзадачами.

#### Scenario: Expand task to see steps

- **WHEN** пользователь раскрывает задачу имеющую шаги
- **THEN** система отображает список всех шагов под родительской задачей с визуальным отступом

#### Scenario: Collapse task to hide steps

- **WHEN** пользователь сворачивает задачу
- **THEN** система скрывает все шаги этой задачи

### Requirement: View total time on task

Система ДОЛЖНА отображать суммарное затраченное время на задачу в формате ЧЧ:ММ:СС.

#### Scenario: Display total time on task form

- **WHEN** пользователь открывает форму задачи
- **THEN** система отображает суммарное время всех отрезков работы над этой задачей

#### Scenario: Real-time time update while timer running

- **WHEN** таймер работает на данной задаче
- **THEN** система обновляет отображаемое суммарное время в реальном времени каждую секунду

### Requirement: Task description rich text

Система ДОЛЖНА поддерживать форматирование текста и вставку скриншотов в описании задачи.

#### Scenario: Format text in description

- **WHEN** пользователь использует форматирование (жирный, курсив, списки) в описании
- **THEN** система сохраняет и отображает форматированный текст

#### Scenario: Insert screenshot in description

- **WHEN** пользователь вставляет скриншот в описание задачи
- **THEN** система сохраняет изображение и отображает его в описании задачи

### Requirement: Link to external task system

Система ДОЛЖНА позволять указать ссылку на задачу во внешней системе (Jira, Trello и т.д.).

#### Scenario: Add external link to task

- **WHEN** пользователь вводит URL внешней задачи в соответствующее поле
- **THEN** система сохраняет ссылку

#### Scenario: Open external task link

- **WHEN** пользователь нажимает на ссылку внешней задачи
- **THEN** система открывает URL в новой вкладке браузера

### Requirement: First action field

Система ДОЛЖНА предоставлять поле "Что конкретно сделать первым" для борьбы с прокрастинацией.

#### Scenario: Set first action

- **WHEN** пользователь заполняет поле "Что конкретно сделать первым"
- **THEN** система сохраняет это значение и отображает его на карточке задачи

#### Scenario: Start task shows first action

- **WHEN** пользователь запускает таймер на задаче с заполненным полем "первое действие"
- **THEN** система отображает напоминание с текстом первого действия

### Requirement: Pomodoro estimate

Система ДОЛЖНА позволять указать оценку трудоёмкости задачи в помодоро.

#### Scenario: Set pomodoro estimate

- **WHEN** пользователь указывает количество помодоро для задачи
- **THEN** система сохраняет оценку

#### Scenario: Display remaining pomodoros

- **WHEN** на задаче отработано N помодоро из оценки M
- **THEN** система отображает "N/M помодоро" на карточке задачи
