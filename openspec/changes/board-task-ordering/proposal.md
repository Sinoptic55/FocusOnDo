## Why

На досках (по дате и по статусу) пользователь может перетягивать задачи между колонками, но не может менять их порядок внутри колонки. Спецификация `task-views` уже содержит сценарий "Manual order preserved", но он не реализован: в модели Task нет поля для сортировки, DragDropManager поддерживает только перемещение между колонками, а бэкенд не имеет эндпоинта для пакетного обновления порядка задач.

## What Changes

- Добавить поле `sort_order` (Integer) в модель Task для хранения пользовательского порядка задач
- Создать миграцию БД для добавления колонки `sort_order` со значением по умолчанию 0
- Добавить API-эндпоинт `PATCH /api/tasks/reorder` для пакетного обновления порядка задач
- Модифицировать DragDropManager для поддержки перетягивания задач внутри одной колонки (определение позиции вставки относительно других задач)
- Обновить компоненты Date Board и Status Board для визуальной индикации точки вставки при перетягивании
- При загрузке задач на доски сортировать по `sort_order` (когда выбрана ручная сортировка)

## Capabilities

### New Capabilities

_Нет новых возможностей._

### Modified Capabilities

- `task-views`: Добавление поддержки ручного порядка задач внутри колонок досок (drag-and-drop reordering). Существующий сценарий "Manual order preserved" получает конкретизацию: определение позиции вставки, сохранение порядка через API, визуальная обратная связь при перетягивании.

## Impact

- **База данных**: Новая колонка `sort_order` в таблице `tasks` (миграция Alembic)
- **Backend API**: Новый эндпоинт `PATCH /api/tasks/reorder`, изменения в схеме `TaskUpdate` и `TaskResponse`
- **Frontend models**: Добавление поля `sort_order` в интерфейс `Task`
- **Frontend drag-drop**: Значительные изменения в `src/frontend/src/utils/drag-drop.ts` для поддержки определения позиции вставки
- **Frontend views**: Изменения в `date-board-view.ts` и `status-board-view.ts` для сортировки по `sort_order` и визуальной индикации
- **API (api.ts)**: Добавление метода для вызова нового эндпоинта reorder
