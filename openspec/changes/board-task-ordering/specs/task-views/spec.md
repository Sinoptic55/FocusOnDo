## MODIFIED Requirements

### Requirement: Drag and drop tasks between views

Система ДОЛЖНА позволять перетаскивать задачи между колонками для изменения их свойств, а также внутри колонки для изменения порядка задач.

#### Scenario: Drag task to different date column

- **WHEN** пользователь перетаскивает задачу из колонки "Сегодня" в колонку "Завтра" на доске по дате
- **THEN** система обновляет planned_date задачи на завтрашнюю дату

#### Scenario: Drag task to different status column

- **WHEN** пользователь перетаскивает задачу из колонки "В работе" в колонку "Завершено" на доске по статусу
- **THEN** система обновляет статус задачи на "Завершено"

#### Scenario: Drag task to different list

- **WHEN** пользователь перетаскивает задачу между списками в виде "Списки"
- **THEN** система обновляет принадлежность задачи к новому списку

#### Scenario: Drag task to reorder within column

- **WHEN** пользователь перетаскивает задачу внутри одной колонки на позицию между двумя другими задачами
- **THEN** система обновляет sort_order перетаскиваемой задачи и смежных задач так, чтобы отразить новый порядок

#### Scenario: Drop position indicator

- **WHEN** пользователь перетягивает задачу над другой задачей в колонке
- **THEN** система отображает визуальный индикатор (линию) выше или ниже целевой задачи, указывая точку вставки

#### Scenario: Drop at end of column

- **WHEN** пользователь перетягивает задачу в пустое пространство колонки ниже всех задач
- **THEN** система помещает задачу в конец колонки

#### Scenario: Cross-column drop with position

- **WHEN** пользователь перетягивает задачу из одной колонки в конкретную позицию внутри другой колонки
- **THEN** система обновляет свойство задачи (planned_date/status_id) и устанавливает sort_order для размещения на указанной позиции

### Requirement: Sort tasks within view

Система ДОЛЖНА позволять сортировать задачи внутри групп/колонок.

#### Scenario: Sort by deadline

- **WHEN** пользователь выбирает сортировку по дедлайну
- **THEN** задачи в каждой колонке отсортированы по возрастанию даты дедлайна

#### Scenario: Sort by priority

- **WHEN** пользователь выбирает сортировку по приоритету
- **THEN** задачи в каждой колонке отсортированы по убыванию приоритета

#### Scenario: Manual order preserved

- **WHEN** пользователь выбирает сортировку "По умолчанию" и вручную перетаскивает задачи для изменения их порядка
- **THEN** система сохраняет пользовательский порядок задач и отображает их по полю sort_order

#### Scenario: Manual order persists after reload

- **WHEN** пользователь изменил порядок задач вручную и перезагрузил страницу
- **THEN** система восстанавливает сохранённый порядок задач из sort_order

## ADDED Requirements

### Requirement: Task reorder API

Система ДОЛЖНА предоставлять API для пакетного обновления порядка задач.

#### Scenario: Batch reorder tasks

- **WHEN** клиент отправляет `PATCH /api/tasks/reorder` с массивом `{task_id, sort_order}`
- **THEN** система обновляет sort_order указанных задач и возвращает подтверждение

#### Scenario: Reorder validates ownership

- **WHEN** клиент отправляет reorder-запрос с task_id, не принадлежащим текущему пользователю
- **THEN** система возвращает ошибку 403 Forbidden

#### Scenario: Reorder validates task_ids exist

- **WHEN** клиент отправляет reorder-запрос с несуществующим task_id
- **THEN** система возвращает ошибку 404 Not Found
