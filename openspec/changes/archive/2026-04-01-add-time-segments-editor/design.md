## Context

В приложении уже существует компонент `TimeSegmentsList` (`src/frontend/src/components/time-segments-list.ts`), который отображает сегменты времени задачи с базовыми CRUD-операциями. Однако он показывает не все поля модели `TimeSegment` — отсутствуют `task_progressed`, `stuck`, `duration_seconds`. Также нет отдельного диалога, который можно открыть из формы задачи для полного управления сегментами.

Форма задачи (`TaskForm`) — это диалог, который открывается для создания/редактирования задачи. В ней есть секция "Основное" с заголовком `<h3>Основное</h3>`.

## Goals / Non-Goals

**Goals:**
- Добавить кнопку-иконку рядом с заголовком "Основное" в TaskForm
- Создать новый диалог `TimeSegmentsEditorDialog`, который отображает все поля time_segments
- Обеспечить CRUD-операции: добавление, редактирование inline, удаление с подтверждением
- Использовать существующие CSS-классы и паттерны приложения для единообразия

**Non-Goals:**
- Изменение бэкенд API — используются существующие эндпоинты
- Изменение модели данных — все поля уже существуют
- Валидация на бэкенде — уже реализована

## Decisions

1. **Создать отдельный компонент `TimeSegmentsEditorDialog`** вместо модификации `TimeSegmentsList`. Причина: `TimeSegmentsList` используется в других контекстах (аналитика), а для формы задачи нужен именно диалог с полным набором полей.

2. **Расширить существующий `TimeSegmentsList`** для отображения всех полей, а не дублировать логику. Новый диалог будет использовать `TimeSegmentsList` как подкомпонент или скопирует его логику с расширениями.

3. **Кнопка — иконка часов (⏱)** рядом с `<h3>Основное</h3>`. Использует класс `btn-icon` для единообразия с другими кнопками в приложении.

4. **Таблица в диалоге** — использует те же CSS-классы `data-table`, `form-control`, `btn-icon`, что и остальное приложение.

## Risks / Trade-offs

- [Дублирование кода между TimeSegmentsList и новым диалогом] → Вынести общую логику в базовый класс или хук
- [Большая таблица при множестве сегментов] → Добавить прокрутку в диалоге (max-height + overflow)
- [Потеря фокуса при inline-редактировании] → Использовать тот же подход, что в TimeSegmentsList (перерисовка только при сохранении)
