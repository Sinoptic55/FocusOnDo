## ADDED Requirements

### Requirement: Theme selection

Система ДОЛЖНА позволять выбрать тему оформления интерфейса.

#### Scenario: Select light theme

- **WHEN** пользователь выбирает светлую тему в настройках
- **THEN** система применяет светлую цветовую схему ко всему интерфейсу

#### Scenario: Select dark theme

- **WHEN** пользователь выбирает тёмную тему в настройках
- **THEN** система применяет тёмную цветовую схему ко всему интерфейсу

#### Scenario: Auto theme based on system

- **WHEN** пользователь выбирает "Автоматически" в настройках темы
- **THEN** система использует тему соответствующую системным настройкам ОС

#### Scenario: Theme persists across sessions

- **WHEN** пользователь выбирает тему и перезагружает приложение
- **THEN** выбранная тема остаётся активной

### Requirement: Sound notifications

Система ДОЛЖНА предоставлять настройки звуковых уведомлений для завершения интервалов.

#### Scenario: Enable work interval notification

- **WHEN** пользователь включает звук завершения работы
- **THEN** система воспроизводит звук при окончании каждого рабочего интервала

#### Scenario: Disable break interval notification

- **WHEN** пользователь отключает звук завершения отдыха
- **THEN** система не воспроизводит звук при окончании интервалов отдыха

#### Scenario: Select notification sound

- **WHEN** пользователь выбирает другой звук из списка доступных
- **THEN** система использует выбранный звук для уведомлений

#### Scenario: Test notification sound

- **WHEN** пользователь нажимает кнопку "Прослушать" рядом с выбором звука
- **THEN** система воспроизводит выбранный звук один раз

#### Scenario: Separate sounds for work and break

- **WHEN** пользователь настраивает звуки
- **THEN** доступны отдельные настройки для завершения работы и завершения отдыха

### Requirement: Hotkey configuration

Система ДОЛЖНА позволять настроить горячую клавишу для быстрого захвата задач.

#### Scenario: Set custom hotkey

- **WHEN** пользователь устанавливает комбинацию Ctrl+Shift+T в настройках
- **THEN** система сохраняет настройку и быстрый захват открывается по этой комбинации

#### Scenario: Default hotkey is Ctrl+Alt+N

- **WHEN** пользователь впервые открывает приложение
- **THEN** горячая клавиша установлена на Ctrl+Alt+N по умолчанию

#### Scenario: Prevent conflicting hotkeys

- **WHEN** пользователь вводит комбинацию уже используемую системой
- **THEN** система предупреждает "Эта комбинация может конфликтовать с системными функциями"

#### Scenario: Record hotkey by pressing keys

- **WHEN** пользователь нажимает в поле ввода горячей клавиши и нажимает комбинацию
- **THEN** система записывает нажатую комбинацию

### Requirement: Morning ritual toggle

Система ДОЛЖНА позволять включать или отключать утренний ритуал планирования.

#### Scenario: Enable morning ritual

- **WHEN** пользователь включает переключатель "Утренний ритуал"
- **THEN** система показывает экран планирования при первом запуске каждого дня

#### Scenario: Disable morning ritual

- **WHEN** пользователь отключает переключатель "Утренний ритуал"
- **THEN** система не показывает экран планирования при запуске

### Requirement: Weekly review day selection

Система ДОЛЖНА позволять выбрать день недели для напоминания о недельном обзоре.

#### Scenario: Set review day to Sunday

- **WHEN** пользователь выбирает воскресенье как день недельного обзора
- **THEN** система показывает экран обзора в первый запуск в воскресенье

#### Scenario: Change review day

- **WHEN** пользователь меняет день обзора с воскресенья на пятницу
- **THEN** следующий обзор будет показан в пятницу

#### Scenario: Disable weekly review

- **WHEN** пользователь выбирает "Не напоминать" для недельного обзора
- **THEN** система не показывает экран обзора автоматически

### Requirement: Stuck detection threshold

Система ДОЛЖНА позволять настроить порог срабатывания детекции застревания.

#### Scenario: Set threshold to 3 intervals

- **WHEN** пользователь устанавливает порог на 3 интервала
- **THEN** уведомление о застревании появляется после 3 последовательных ответов "Застрял"

#### Scenario: Set threshold to 5 intervals

- **WHEN** пользователь устанавливает порог на 5 интервалов
- **THEN** уведомление о застревании появляется только после 5 последовательных ответов "Застрял"

#### Scenario: Default threshold is 3

- **WHEN** пользователь впервые использует систему
- **THEN** порог детекции застревания установлен на 3 интервала

### Requirement: Auto-start next interval setting

Система ДОЛЖНА позволять включить автоматический запуск следующего интервала.

#### Scenario: Enable auto-start

- **WHEN** пользователь включает "Автостарт следующего интервала"
- **THEN** таймер автоматически запускает следующий интервал через 3 секунды после завершения текущего

#### Scenario: Disable auto-start

- **WHEN** пользователь отключает автостарт
- **THEN** таймер ждёт ручного нажатия "Пуск" после завершения интервала

### Requirement: Language selection

Система ДОЛЖНА позволять выбрать язык интерфейса.

#### Scenario: Select English language

- **WHEN** пользователь выбирает английский язык в настройках
- **THEN** весь интерфейс отображается на английском

#### Scenario: Select Russian language

- **WHEN** пользователь выбирает русский язык в настройках
- **THEN** весь интерфейс отображается на русском

#### Scenario: Default language based on browser

- **WHEN** пользователь впервые открывает приложение
- **THEN** язык интерфейса соответствует языку браузера (если поддерживается)

### Requirement: Notification persistence

Система ДОЛЖНА позволять настроить настойчивость уведомлений.

#### Scenario: Enable persistent notifications

- **WHEN** пользователь включает "Настойчивые уведомления"
- **THEN** уведомление о завершении интервала остаётся на экране до взаимодействия

#### Scenario: Disable persistent notifications

- **WHEN** пользователь отключает настойчивые уведомления
- **THEN** уведомление автоматически исчезает через 10 секунд

### Requirement: Time format preference

Система ДОЛЖНА позволять выбрать формат отображения времени.

#### Scenario: Select 24-hour format

- **WHEN** пользователь выбирает 24-часовой формат
- **THEN** время отображается как 14:30 вместо 2:30 PM

#### Scenario: Select 12-hour format

- **WHEN** пользователь выбирает 12-часовой формат
- **THEN** время отображается с AM/PM

### Requirement: Settings persistence

Система ДОЛЖНА сохранять все настройки пользователя.

#### Scenario: Save settings to server

- **WHEN** пользователь изменяет любую настройку
- **THEN** система сохраняет настройку на сервер в app_settings.settings_json

#### Scenario: Restore settings on login

- **WHEN** пользователь входит в систему с другого устройства
- **THEN** все его настройки загружаются и применяются

#### Scenario: Settings validation

- **WHEN** пользователь вводит некорректное значение настройки
- **THEN** система отображает ошибку валидации и не сохраняет изменение

### Requirement: Reset to defaults

Система ДОЛЖНА позволять сбросить все настройки к значениям по умолчанию.

#### Scenario: Reset all settings

- **WHEN** пользователь нажимает "Сбросить все настройки" и подтверждает действие
- **THEN** система восстанавливает все настройки к начальным значениям

#### Scenario: Confirm before reset

- **WHEN** пользователь нажимает "Сбросить все настройки"
- **THEN** система запрашивает подтверждение "Вы уверены? Это действие нельзя отменить"

### Requirement: Settings categories

Система ДОЛЖНА группировать настройки по категориям для удобства навигации.

#### Scenario: Navigate between settings sections

- **WHEN** пользователь открывает страницу настроек
- **THEN** настройки разделены на секции: Внешний вид, Уведомления, Умные функции, Таймер, Язык и регион

#### Scenario: Direct link to settings section

- **WHEN** пользователь переходит по ссылке на конкретную секцию настроек
- **THEN** страница открывается с прокруткой к этой секции
