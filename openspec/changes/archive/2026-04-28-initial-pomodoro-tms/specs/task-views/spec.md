## ADDED Requirements

### Requirement: Lists view

Система ДОЛЖНА предоставлять вид отображения задач сгруппированных по спискам.

#### Scenario: Display tasks grouped by lists

- **WHEN** пользователь открывает вид "Списки"
- **THEN** система отображает задачи сгруппированные по их спискам (Работа, Личное, и т.д.)

#### Scenario: Tasks show list color indicator

- **WHEN** задача отображается в любом виде
- **THEN** задача имеет цветную полоску соответствующую цвету её списка

#### Scenario: Empty list shows placeholder

- **WHEN** список не содержит задач
- **THEN** система отображает пустой список с текстом "Нет задач"

### Requirement: Date board view

Система ДОЛЖНА предоставлять вид отображения задач в виде доски с колонками по датам.

#### Scenario: Display six date columns

- **WHEN** пользователь открывает вид "Доска по дате"
- **THEN** система отображает 6 колонок: "Без срока", "Сегодня", "Завтра", "Ближайшая неделя", "Позже", "Когда-нибудь"

#### Scenario: Task in Today column based on planned date

- **WHEN** у задачи planned_date равна сегодняшней дате
- **THEN** задача отображается в колонке "Сегодня"

#### Scenario: Task in Tomorrow column

- **WHEN** у задачи planned_date равна завтрашней дате
- **THEN** задача отображается в колонке "Завтра"

#### Scenario: Task in This Week column

- **WHEN** у задачи planned_date в пределах текущей недели (но не сегодня и не завтра)
- **THEN** задача отображается в колонке "Ближайшая неделя"

#### Scenario: Task in Later column

- **WHEN** у задачи planned_date больше текущей недели но меньше 30 дней
- **THEN** задача отображается в колонке "Позже"

#### Scenario: Task in Someday column

- **WHEN** у задачи planned_date больше 30 дней в будущем
- **THEN** задача отображается в колонке "Когда-нибудь"

#### Scenario: Task in No Date column

- **WHEN** у задачи не указан planned_date
- **THEN** задача отображается в колонке "Без срока"

### Requirement: Status board view

Система ДОЛЖНА предоставлять вид отображения задач в виде доски с колонками по статусам.

#### Scenario: Display status columns

- **WHEN** пользователь открывает вид "Доска по статусу"
- **THEN** система отображает колонки для каждого статуса с флагом board_visible=true

#### Scenario: Tasks grouped by status

- **WHEN** задачи имеют различные статусы
- **THEN** каждая задача отображается в колонке соответствующей её статусу

#### Scenario: Hidden status not shown on board

- **WHEN** статус имеет флаг board_visible=false
- **THEN** колонка для этого статуса не отображается на доске

### Requirement: Task card visual presentation

Система ДОЛЖНА отображать карточку задачи с визуальными индикаторами.

#### Scenario: Task card shows list color border

- **WHEN** задача отображается на доске
- **THEN** карточка задачи имеет цветную рамку соответствующую цвету списка

#### Scenario: Task card shows project color label

- **WHEN** задача принадлежит проекту
- **THEN** карточка отображает цветной ярлык с названием проекта

#### Scenario: Task card shows pomodoro estimate

- **WHEN** у задачи указана оценка в помодоро
- **THEN** карточка отображает иконку помидора с числом оценки

#### Scenario: Task card shows deadline indicator

- **WHEN** у задачи есть дедлайн
- **THEN** карточка отображает дату дедлайна с цветовым индикатором срочности (красный если просрочено, оранжевый если скоро)

### Requirement: Drag and drop tasks between views

Система ДОЛЖНА позволять перетаскивать задачи между колонками для изменения их свойств.

#### Scenario: Drag task to different date column

- **WHEN** пользователь перетаскивает задачу из колонки "Сегодня" в колонку "Завтра" на доске по дате
- **THEN** система обновляет planned_date задачи на завтрашнюю дату

#### Scenario: Drag task to different status column

- **WHEN** пользователь перетаскивает задачу из колонки "В работе" в колонку "Завершено" на доске по статусу
- **THEN** система обновляет статус задачи на "Завершено"

#### Scenario: Drag task to different list

- **WHEN** пользователь перетаскивает задачу между списками в виде "Списки"
- **THEN** система обновляет принадлежность задачи к новому списку

### Requirement: View switching persistence

Система ДОЛЖНА запоминать последний выбранный вид отображения.

#### Scenario: Remember selected view

- **WHEN** пользователь переключается на вид "Доска по дате" и перезагружает страницу
- **THEN** система открывает вид "Доска по дате" а не вид по умолчанию

### Requirement: Filter tasks in views

Система ДОЛЖНА позволять фильтровать задачи в любом виде отображения.

#### Scenario: Filter by project

- **WHEN** пользователь выбирает фильтр по проекту
- **THEN** система отображает только задачи выбранного проекта

#### Scenario: Filter by client

- **WHEN** пользователь выбирает фильтр по клиенту
- **THEN** система отображает только задачи выбранного клиента

#### Scenario: Clear filters

- **WHEN** пользователь нажимает "Очистить фильтры"
- **THEN** система отображает все задачи без фильтрации

### Requirement: Sort tasks within view

Система ДОЛЖНА позволять сортировать задачи внутри групп/колонок.

#### Scenario: Sort by deadline

- **WHEN** пользователь выбирает сортировку по дедлайну
- **THEN** задачи в каждой колонке отсортированы по возрастанию даты дедлайна

#### Scenario: Sort by priority

- **WHEN** пользователь выбирает сортировку по приоритету
- **THEN** задачи в каждой колонке отсортированы по убыванию приоритета

#### Scenario: Manual order preserved

- **WHEN** пользователь вручную перетаскивает задачи для изменения их порядка
- **THEN** система сохраняет пользовательский порядок задач
