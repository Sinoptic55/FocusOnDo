## ADDED Requirements

### Requirement: Quick task capture with hotkey

Система ДОЛЖНА позволять быстро захватывать задачи через мини-окно вызываемое горячей клавишей.

#### Scenario: Open quick capture with hotkey

- **WHEN** пользователь нажимает настроенную горячую клавишу (по умолчанию Ctrl+Alt+N) из любой программы
- **THEN** система открывает мини-окно поверх всех окон для ввода задачи

#### Scenario: Create task from quick capture

- **WHEN** пользователь вводит текст в мини-окно и нажимает Enter
- **THEN** система создаёт задачу с этим названием в списке "Входящие" (Inbox)

#### Scenario: Quick capture does not interrupt timer

- **WHEN** пользователь открывает quick capture во время работы таймера
- **THEN** таймер продолжает работать без прерывания

#### Scenario: Close quick capture without saving

- **WHEN** пользователь нажимает Escape в мини-окне
- **THEN** окно закрывается без создания задачи

### Requirement: Auto-select priority task

Система ДОЛЖНА предоставлять функцию автоматического выбора приоритетной задачи.

#### Scenario: Select overdue task first

- **WHEN** пользователь нажимает "Не знаю, с чего начать" и есть просроченные задачи (дедлайн прошёл)
- **THEN** система выбирает задачу с самым ранним просроченным дедлайном

#### Scenario: Select today task when no overdue

- **WHEN** нет просроченных задач но есть задачи на сегодня
- **THEN** система выбирает задачу на сегодня с наивысшим приоритетом

#### Scenario: Select any task when no urgent

- **WHEN** нет просроченных задач и задач на сегодня
- **THEN** система выбирает первую задачу из ближайших по planned_date

#### Scenario: Start timer on selected task

- **WHEN** система выбирает приоритетную задачу
- **THEN** задача автоматически устанавливается как активная для таймера

#### Scenario: Show selection reason

- **WHEN** система автоматически выбирает задачу
- **THEN** отображается уведомление с причиной (например "Просрочен дедлайн 3 дня назад")

### Requirement: Morning ritual planning screen

Система ДОЛЖНА показывать экран планирования дня при первом запуске за день.

#### Scenario: Display morning ritual on first launch

- **WHEN** пользователь открывает приложение впервые за текущий день
- **THEN** система отображает экран утреннего ритуала

#### Scenario: Show urgent tasks on ritual screen

- **WHEN** открывается экран утреннего ритуала
- **THEN** система отображает блок с просроченными задачами и задачами на сегодня

#### Scenario: Show inbox tasks count

- **WHEN** открывается экран утреннего ритуала и во "Входящих" есть задачи
- **THEN** система отображает количество задач к разбору во Входящих

#### Scenario: Drag tasks to Today

- **WHEN** пользователь перетаскивает задачу на область "Сегодня" на экране ритуала
- **THEN** система устанавливает planned_date задачи на сегодня

#### Scenario: Skip morning ritual

- **WHEN** пользователь нажимает "Пропустить" на экране утреннего ритуала
- **THEN** система закрывает экран и не показывает его до следующего дня

#### Scenario: Disable morning ritual in settings

- **WHEN** пользователь отключает утренний ритуал в настройках
- **THEN** система не показывает экран планирования при запуске

### Requirement: Weekly review

Система ДОЛЖНА предлагать недельный обзор в настроенный день недели.

#### Scenario: Trigger weekly review on set day

- **WHEN** наступает настроенный день недели (например, воскресенье) и пользователь открывает приложение
- **THEN** система отображает экран недельного обзора

#### Scenario: Show completed tasks summary

- **WHEN** открывается недельный обзор
- **THEN** система отображает количество завершённых задач за неделю

#### Scenario: Show incomplete tasks

- **WHEN** открывается недельный обзор
- **THEN** система отображает список незавершённых задач с истёкшим planned_date

#### Scenario: Show time statistics

- **WHEN** открывается недельный обзор
- **THEN** система отображает суммарное время работы за неделю по проектам

#### Scenario: Move task to next week

- **WHEN** пользователь выбирает "Перенести на следующую неделю" для задачи
- **THEN** система обновляет planned_date на соответствующий день следующей недели

#### Scenario: Archive completed tasks

- **WHEN** пользователь нажимает "Архивировать завершённые" на экране обзора
- **THEN** система архивирует все завершённые задачи старше недели

#### Scenario: Postpone review

- **WHEN** пользователь нажимает "Напомнить завтра"
- **THEN** система не показывает обзор до следующего дня

### Requirement: Stuck pattern detection

Система ДОЛЖНА обнаруживать паттерн застревания на задаче и предлагать разбить её.

#### Scenario: Detect multiple stuck responses

- **WHEN** задача получает ответ "Застрял" в 3 последовательных интервалах
- **THEN** система отображает уведомление "Похоже вы застряли на этой задаче"

#### Scenario: Suggest breaking into steps

- **WHEN** обнаружен паттерн застревания
- **THEN** система предлагает "Разбить задачу на более мелкие шаги?"

#### Scenario: Open steps editor from suggestion

- **WHEN** пользователь принимает предложение разбить задачу
- **THEN** система открывает редактор задачи с фокусом на создании шагов

#### Scenario: Configurable stuck threshold

- **WHEN** пользователь изменяет порог детекции застревания в настройках на 5 интервалов
- **THEN** уведомление появляется только после 5 ответов "Застрял"

#### Scenario: Dismiss stuck notification

- **WHEN** пользователь закрывает уведомление о застревании
- **THEN** система не показывает его снова для этой задачи в текущей сессии

### Requirement: Inbox processing workflow

Система ДОЛЖНА предоставлять удобный процесс разбора входящих задач.

#### Scenario: Enter inbox processing mode

- **WHEN** пользователь нажимает "Разобрать входящие"
- **THEN** система отображает первую задачу из Inbox с опциями: Запланировать, Делегировать, Отложить, Удалить

#### Scenario: Schedule inbox task

- **WHEN** пользователь выбирает "Запланировать" для задачи из Inbox
- **THEN** система открывает быстрый выбор даты и перемещает задачу в соответствующий список

#### Scenario: Delete inbox task

- **WHEN** пользователь выбирает "Удалить" для задачи из Inbox
- **THEN** система удаляет задачу и показывает следующую

#### Scenario: Process all inbox items

- **WHEN** пользователь обрабатывает последнюю задачу из Inbox
- **THEN** система отображает сообщение "Входящие пусты!" и закрывает режим обработки

### Requirement: Smart task suggestions

Система ДОЛЖНА предлагать умные рекомендации на основе паттернов работы.

#### Scenario: Suggest best time to work on task

- **WHEN** пользователь просматривает задачу
- **THEN** система анализирует историю energy_level и предлагает "Лучшее время для этой задачи: утро (9-12)"

#### Scenario: Suggest similar tasks

- **WHEN** пользователь создаёт задачу похожую на существующие
- **THEN** система предлагает "Похожие задачи" для справки или связи

#### Scenario: Warn about overestimation

- **WHEN** пользователь планирует на день задач на 16 помодоро (8 часов)
- **THEN** система предупреждает "Обычно вы выполняете 10 помодоро в день"
