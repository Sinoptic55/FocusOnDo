## ADDED Requirements

### Requirement: Manage task lists

Система ДОЛЖНА позволять пользователю создавать и управлять списками задач с цветовой маркировкой.

#### Scenario: Create new task list

- **WHEN** пользователь создаёт новый список с названием и выбранным цветом
- **THEN** система сохраняет список и он становится доступен для привязки задач

#### Scenario: Edit task list

- **WHEN** пользователь изменяет название или цвет списка
- **THEN** система обновляет список и все задачи этого списка отображаются с новым цветом

#### Scenario: Delete task list with tasks

- **WHEN** пользователь пытается удалить список содержащий задачи
- **THEN** система запрашивает подтверждение и предлагает переместить задачи в другой список

#### Scenario: Default task list exists

- **WHEN** пользователь впервые запускает систему
- **THEN** существует список по умолчанию "Входящие" (Inbox)

### Requirement: Manage task statuses

Система ДОЛЖНА позволять пользователю создавать и управлять статусами задач.

#### Scenario: Create new status

- **WHEN** пользователь создаёт новый статус с названием
- **THEN** система сохраняет статус и он становится доступен для задач

#### Scenario: Set status board visibility

- **WHEN** пользователь переключает флаг "Отображать на доске" для статуса
- **THEN** система сохраняет board_visible и статус появляется/исчезает на доске по статусам

#### Scenario: Reorder statuses

- **WHEN** пользователь изменяет порядок статусов
- **THEN** система сохраняет новый порядок и отображает колонки на доске в этом порядке

#### Scenario: Default statuses exist

- **WHEN** пользователь впервые запускает систему
- **THEN** существуют статусы: "Новая", "В работе", "На проверке", "Завершено"

#### Scenario: Cannot delete status with tasks

- **WHEN** пользователь пытается удалить статус с задачами
- **THEN** система отображает ошибку и предлагает сначала переместить задачи

### Requirement: Manage projects

Система ДОЛЖНА позволять пользователю создавать и управлять проектами с цветовой маркировкой.

#### Scenario: Create new project

- **WHEN** пользователь создаёт проект с названием и цветом
- **THEN** система сохраняет проект и он становится доступен для привязки к задачам

#### Scenario: Edit project

- **WHEN** пользователь изменяет название или цвет проекта
- **THEN** система обновляет проект и все задачи отображаются с новым цветным ярлыком

#### Scenario: Archive completed project

- **WHEN** пользователь архивирует проект
- **THEN** проект скрывается из списка активных но задачи сохраняются с привязкой к нему

#### Scenario: View archived projects

- **WHEN** пользователь открывает список архивных проектов
- **THEN** система отображает все архивированные проекты с возможностью восстановления

### Requirement: Manage clients

Система ДОЛЖНА позволять пользователю создавать и управлять заказчиками (клиентами).

#### Scenario: Create new client

- **WHEN** пользователь создаёт клиента с названием
- **THEN** система сохраняет клиента и он становится доступен для привязки к задачам и проектам

#### Scenario: Edit client

- **WHEN** пользователь изменяет название клиента
- **THEN** система обновляет клиента и изменения отражаются на всех связанных задачах

#### Scenario: Delete client without tasks

- **WHEN** пользователь удаляет клиента без привязанных задач
- **THEN** система удаляет клиента

#### Scenario: Cannot delete client with tasks or projects

- **WHEN** пользователь пытается удалить клиента с привязанными задачами или проектами
- **THEN** система отображает ошибку с количеством связанных объектов

### Requirement: Manage timer intervals configuration

Система ДОЛЖНА позволять пользователю настраивать последовательность интервалов таймера.

#### Scenario: View current interval sequence

- **WHEN** пользователь открывает настройки интервалов
- **THEN** система отображает текущую последовательность с типом и длительностью каждого интервала

#### Scenario: Add new interval to sequence

- **WHEN** пользователь добавляет новый интервал указав тип (работа/отдых) и длительность в минутах
- **THEN** система добавляет интервал в конец последовательности

#### Scenario: Edit interval duration

- **WHEN** пользователь изменяет длительность существующего интервала
- **THEN** система сохраняет новую длительность и применяет её при следующем цикле

#### Scenario: Delete interval from sequence

- **WHEN** пользователь удаляет интервал из последовательности
- **THEN** система удаляет интервал и сдвигает остальные интервалы

#### Scenario: Reorder intervals

- **WHEN** пользователь перетаскивает интервалы для изменения порядка
- **THEN** система сохраняет новый порядок последовательности

#### Scenario: Reset to default sequence

- **WHEN** пользователь нажимает "Восстановить по умолчанию"
- **THEN** система возвращает классическую последовательность Помодоро (25-5-25-5-25-5-25-15)

### Requirement: Color picker for lists and projects

Система ДОЛЖНА предоставлять выбор цвета для списков и проектов.

#### Scenario: Display color palette

- **WHEN** пользователь создаёт или редактирует список или проект
- **THEN** система отображает палитру с предустановленными цветами

#### Scenario: Select color from palette

- **WHEN** пользователь выбирает цвет из палитры
- **THEN** система применяет выбранный цвет к списку или проекту

#### Scenario: Custom color input

- **WHEN** пользователь вводит hex код цвета вручную
- **THEN** система применяет указанный цвет

### Requirement: Reference data isolation per user

Система ДОЛЖНА изолировать справочные данные между пользователями.

#### Scenario: User sees only own reference data

- **WHEN** пользователь просматривает списки, статусы, проекты или клиентов
- **THEN** система отображает только объекты созданные этим пользователем

#### Scenario: Reference data not shared

- **WHEN** пользователь A создаёт проект
- **THEN** пользователь B не видит этот проект в своём списке

### Requirement: Validate reference data

Система ДОЛЖНА валидировать справочные данные при создании и редактировании.

#### Scenario: Unique list names

- **WHEN** пользователь пытается создать список с именем уже существующего
- **THEN** система отображает ошибку "Список с таким именем уже существует"

#### Scenario: Unique status names

- **WHEN** пользователь пытается создать статус с именем уже существующего
- **THEN** система отображает ошибку "Статус с таким именем уже существует"

#### Scenario: Required name fields

- **WHEN** пользователь пытается сохранить список, проект или клиента без имени
- **THEN** система отображает ошибку "Название обязательно"
