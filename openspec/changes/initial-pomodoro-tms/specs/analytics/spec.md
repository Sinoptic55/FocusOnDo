## ADDED Requirements

### Requirement: Time by projects report

Система ДОЛЖНА предоставлять отчёт по времени потраченному на проекты.

#### Scenario: Display total time per project

- **WHEN** пользователь открывает отчёт "Время по проектам"
- **THEN** система отображает каждый проект с суммарным фактическим и выставленным временем

#### Scenario: Filter time report by date range

- **WHEN** пользователь выбирает период "Последний месяц"
- **THEN** отчёт показывает только время отработанное в последние 30 дней

#### Scenario: Show actual vs billed time difference

- **WHEN** отображается отчёт по проектам
- **THEN** для каждого проекта показывается разница (Фактическое - Выставленное) время

#### Scenario: Export project time report

- **WHEN** пользователь нажимает "Экспорт" на отчёте
- **THEN** система генерирует CSV файл с данными отчёта

### Requirement: Time by clients report

Система ДОЛЖНА предоставлять отчёт по времени потраченному на клиентов.

#### Scenario: Display total time per client

- **WHEN** пользователь открывает отчёт "Время по клиентам"
- **THEN** система отображает каждого клиента с суммарным временем всех его задач

#### Scenario: Drill down to client projects

- **WHEN** пользователь раскрывает клиента в отчёте
- **THEN** система показывает детализацию по проектам этого клиента

#### Scenario: Show billable amount estimate

- **WHEN** для клиента/проекта настроена почасовая ставка
- **THEN** отчёт отображает расчётную сумму к выставлению на основе billed_time

### Requirement: Productivity peaks analytics

Система ДОЛЖНА анализировать и отображать пики продуктивности по времени дня.

#### Scenario: Display energy level by hour

- **WHEN** пользователь открывает отчёт "Пики продуктивности"
- **THEN** система отображает график распределения уровня энергии по часам дня

#### Scenario: Identify best working hours

- **WHEN** накоплено достаточно данных (минимум 20 рабочих интервалов)
- **THEN** система выделяет часы с наивысшим процентом ответов "Высокая энергия"

#### Scenario: Show task completion rate by time

- **WHEN** отображается график продуктивности
- **THEN** включается метрика процента интервалов с ответом "Задача продвинулась" по часам

#### Scenario: Recommend work schedule

- **WHEN** определены пики продуктивности
- **THEN** система рекомендует "Ваше лучшее время для сложных задач: 9:00-12:00 и 15:00-17:00"

### Requirement: Estimation accuracy analytics

Система ДОЛЖНА анализировать точность оценок задач в помодоро.

#### Scenario: Compare estimate vs actual pomodoros

- **WHEN** пользователь открывает отчёт "Точность оценки"
- **THEN** система отображает завершённые задачи с оценкой и фактическим количеством помодоро

#### Scenario: Calculate average estimation error

- **WHEN** отображается отчёт точности оценки
- **THEN** система показывает среднее отклонение: "+15% (переоцениваете сложность)"

#### Scenario: Show estimation trend

- **WHEN** накоплена история за несколько месяцев
- **THEN** отчёт отображает график динамики точности оценки по времени

#### Scenario: Filter by project or task type

- **WHEN** пользователь выбирает фильтр по проекту
- **THEN** анализ точности показывается только для задач этого проекта

### Requirement: Work speed analytics

Система ДОЛЖНА отслеживать скорость работы (задачи в день/неделю).

#### Scenario: Display tasks completed per day

- **WHEN** пользователь открывает отчёт "Скорость работы"
- **THEN** система отображает среднее количество завершённых задач в день

#### Scenario: Show weekly completion trend

- **WHEN** отображается отчёт скорости работы
- **THEN** система показывает график количества завершённых задач по неделям

#### Scenario: Compare current vs previous period

- **WHEN** отображается отчёт за текущую неделю
- **THEN** система показывает сравнение с предыдущей неделей: "На 20% больше задач"

#### Scenario: Average pomodoros per task

- **WHEN** отображается отчёт скорости
- **THEN** система показывает среднее количество помодоро на одну завершённую задачу

### Requirement: Stuck patterns analytics

Система ДОЛЖНА анализировать паттерны застревания на задачах.

#### Scenario: List tasks with high stuck rate

- **WHEN** пользователь открывает отчёт "Паттерны застревания"
- **THEN** система отображает задачи с наибольшим процентом интервалов с ответом "Застрял"

#### Scenario: Identify common stuck factors

- **WHEN** отображается анализ застревания
- **THEN** система группирует по факторам: тип задачи, проект, время дня, сложность

#### Scenario: Show stuck resolution time

- **WHEN** задача переходит из состояния "застревания" к прогрессу
- **THEN** система фиксирует время разблокировки и включает в статистику

#### Scenario: Suggest actionable improvements

- **WHEN** обнаружен паттерн застревания на определённом типе задач
- **THEN** система предлагает "Задачи типа X часто вызывают трудности после 16:00. Попробуйте делать их утром"

### Requirement: Dashboard overview

Система ДОЛЖНА предоставлять dashboard с ключевыми метриками.

#### Scenario: Display today summary

- **WHEN** пользователь открывает dashboard
- **THEN** система отображает метрики за сегодня: помодоро отработано, задач завершено, текущая задача, суммарное время

#### Scenario: Display week summary

- **WHEN** отображается dashboard
- **THEN** включается блок с метриками недели: всего времени, завершённых задач, топ-3 проекта по времени

#### Scenario: Show upcoming deadlines

- **WHEN** отображается dashboard
- **THEN** система показывает ближайшие 5 дедлайнов с индикацией срочности

#### Scenario: Quick access to reports

- **WHEN** пользователь нажимает на метрику в dashboard
- **THEN** система открывает соответствующий детальный отчёт

### Requirement: Custom date range for all reports

Система ДОЛЖНА позволять выбирать произвольный период для всех отчётов.

#### Scenario: Select custom date range

- **WHEN** пользователь выбирает "Произвольный период" и указывает даты с 1 по 15 марта
- **THEN** все отчёты отображают данные только за указанный период

#### Scenario: Preset periods available

- **WHEN** пользователь открывает выбор периода
- **THEN** доступны пресеты: Сегодня, Вчера, Последние 7 дней, Последние 30 дней, Текущая неделя, Текущий месяц

### Requirement: Visual charts for analytics

Система ДОЛЖНА отображать данные аналитики в виде графиков.

#### Scenario: Bar chart for project time

- **WHEN** отображается отчёт "Время по проектам"
- **THEN** данные представлены в виде столбчатой диаграммы с цветами проектов

#### Scenario: Line chart for trends

- **WHEN** отображается тренд завершённых задач по неделям
- **THEN** используется линейный график

#### Scenario: Pie chart for time distribution

- **WHEN** пользователь просматривает распределение времени между проектами
- **THEN** доступна круговая диаграмма с процентами

### Requirement: Compare periods

Система ДОЛЖНА позволять сравнивать метрики между периодами.

#### Scenario: Compare this week vs last week

- **WHEN** пользователь выбирает режим сравнения "Текущая неделя vs Прошлая"
- **THEN** все метрики отображаются с процентом изменения

#### Scenario: Highlight improvements and declines

- **WHEN** отображается сравнение периодов
- **THEN** улучшения показываются зелёным, ухудшения красным цветом
