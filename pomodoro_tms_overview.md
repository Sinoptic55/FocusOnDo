# Pomodoro TMS — Описание системы

## Planned Technology Stack

- **Frontend**: TypeScript, JavaScript, HTML
- **Backend**: Python (simple web server)
- **Database**: PostgreSQL
- **Authentication**: Login/password
- **Data Storage**: Server-side with internet access
- **Development Principle**: KISS (Keep It Simple, Stupid)

---

## Что это такое

Приложение для управления задачами и учёта рабочего времени. Оно объединяет два подхода: классические списки задач и визуальные доски, а в центре всего — таймер Помодоро, который помогает работать сосредоточенно и автоматически фиксирует, сколько времени ушло на каждую задачу.

Приложение решает сразу две задачи: личную продуктивность (не прокрастинировать, понимать куда уходит время) и деловую (знать сколько времени потрачено на проект или клиента, сколько выставить в счёте).

---

## Для кого

На первом этапе — для одного пользователя. В будущем планируется расширение для небольших команд.

---

## Как выглядит приложение

Приложение доступно через браузер и как отдельная программа для Windows. В перспективе появится мобильная версия.

Внизу экрана всегда виден таймер Помодоро — он не исчезает при переходе между разделами. Это главный элемент интерфейса.

## Задачи

### Структура задач

Задачи организованы иерархически: у каждой задачи могут быть подзадачи — они называются шагами. Это позволяет разбивать большую работу на конкретные действия.

У каждой задачи есть:
- Название
- Описание (с форматированием текста и вставкой скриншотов)
- Дата, на которую задача запланирована
- Крайний срок (дедлайн)
- Статус (например: Новая, В работе, На проверке, Завершено)
- Принадлежность к проекту и заказчику
- Оценка трудоёмкости в помодоро
- Поле «Что конкретно сделать первым» — помогает начать без промедления
- Ссылка на задачу во внешней системе (Jira, Trello и т.д.) — для справки
- Суммарное затраченное время в формате ЧЧ:ММ:СС

### Отображение задач

Задачи можно смотреть тремя способами:

**Списки** — задачи сгруппированы по спискам (например: Работа, Личное, Проект X). Каждый список имеет свой цвет — задачи визуально выделяются цветной полоской.

**Доска по дате** — задачи разложены по колонкам: Без срока, Сегодня, Завтра, Ближайшая неделя, Позже, Когда-нибудь.

**Доска по статусу** — задачи разложены по статусам. Колонки настраиваются.

На досках карточка задачи обведена цветом её списка. Проект отображается цветным ярлыком.

## Таймер Помодоро

Техника Помодоро — это чередование рабочих и отдыхательных интервалов. Классический вариант: 25 минут работы, 5 минут отдыха, после 4 циклов — длинный перерыв 15 минут.

### Как работает таймер

Таймер всегда знает, над какой задачей ты работаешь. Когда идёт рабочий интервал — время автоматически начисляется на текущую задачу.

Кнопки управления:
- **Пуск / Пауза** — запустить или поставить на паузу
- **Следующий** — перейти к следующему интервалу досрочно

Если задача не выбрана — при нажатии «Пуск» приложение предложит выбрать из задач на сегодня.

### Настройка интервалов

Последовательность интервалов настраивается: можно добавлять, удалять, менять порядок и длительность. Каждый интервал — либо «Работа», либо «Отдых».

### После каждого рабочего интервала

Приложение задаёт два быстрых вопроса (можно пропустить):
1. Задача продвинулась? (Да / Нет / Застрял)
2. Какой был уровень энергии? (Высокий / Средний / Низкий)

Это занимает 5 секунд, но со временем даёт ценную картину: в какое время дня ты работаешь лучше всего, какие задачи вызывают затруднения.

### Паттерн застревания

Если задача получает ответ «Застрял» несколько раз подряд — приложение предлагает разбить её на более мелкие шаги. Это работает потому что мозг избегает задач с непонятным следующим действием.

## Учёт времени

Время учитывается автоматически пока идёт рабочий интервал таймера. Каждый отрезок работы фиксируется с точным временем начала и длительностью в секундах.

На форме задачи всегда видно суммарное затраченное время (ЧЧ:ММ:СС), и оно обновляется в реальном времени пока таймер идёт.

### Фактическое и выставленное время

Помимо фактически затраченного времени, каждый отрезок имеет поле «Выставленное время» — то, которое будет учтено в счёте клиенту. Его можно скорректировать вручную. Если не трогать — оно равно фактическому.

## Умные инструменты

### «Не знаю, с чего начать»

Одна кнопка — одна задача. Приложение само выбирает самое важное по приоритету: сначала просроченные дедлайны, потом задачи на сегодня, потом всё остальное. Убирает паралич выбора.

### Быстрый захват

Горячая клавиша (настраивается) открывает мини-окно поверх любой программы. Вводишь мысль одной строкой — она сохраняется в папку «Входящие». Таймер при этом не прерывается. Потом разбираешь входящие в удобное время.

### Планирование дня (утренний ритуал)

При первом запуске приложения за день открывается экран планирования. Показывает что срочно, что накопилось во входящих, и позволяет перетащить нужные задачи в «Сегодня». Занимает 2–3 минуты, но структурирует день.

### Недельный обзор

Раз в неделю — экран с итогами: что сделано, что не сделано, сколько времени куда ушло. Незавершённые задачи можно перенести на следующую неделю, отложить на потом или удалить. Снимает тревогу от «зависших» дел.

## Повторяющиеся задачи

Задачи можно настроить как повторяющиеся: каждый день, каждую неделю (в определённые дни), каждый месяц и так далее. Когда такая задача выполнена — следующая создаётся автоматически со всеми шагами.

---

## Проекты и заказчики

Каждая задача может принадлежать проекту и заказчику. Это позволяет в отчётах видеть: сколько времени ушло на конкретный проект, сколько на клиента, сколько выставить в счёте.

Каждый проект имеет свой цвет — он отображается везде где упоминается проект задачи.

---

## Справочники

Настраиваются через интерфейс:
- **Списки задач** — с цветом
- **Статусы** — с возможностью включать/выключать отображение на доске
- **Проекты** — с цветом
- **Заказчики**
- **Интервалы таймера** — порядок, тип (работа/отдых), длительность

---

## Аналитика

Накапливается с первого дня. Включает:

- **Время по проектам и заказчикам** — фактическое и выставленное
- **Пики продуктивности** — в какое время дня энергия выше всего
- **Точность оценки** — насколько точно ты оцениваешь задачи в помодоро
- **Скорость работы** — сколько задач завершаешь в день/неделю
- **Паттерны застревания** — какие задачи чаще всего вызывают затруднения

## Настройки приложения

- Тёмная и светлая тема (или автоматически по системе)
- Звуковые уведомления об окончании интервалов — раздельно для работы и отдыха, несколько вариантов звуков
- Горячая клавиша для быстрого захвата задач
- Включить / выключить утренний ритуал
- День недели для напоминания о недельном обзоре
- Порог срабатывания уведомления о застревании

## Техническая платформа

- Серверное приложение, работающее на Windows Server
- База данных PostgreSQL
- Веб-интерфейс (браузер) и отдельное приложение для Windows
- В будущем — мобильное приложение
- Все данные хранятся на сервере, доступ через интернет
- Вход по логину и паролю
